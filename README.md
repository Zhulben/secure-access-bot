# Secure Access Telegram Bot

Телеграм-бот с:
- регистрацией по имени и фамилии;
- подтверждением доступа админом;
- баном/разбаном;
- недельными ключами доступа;
- ежедневной активацией доступа на 24 часа;
- хранением фотографий в PostgreSQL (`BYTEA`);
- выдачей последней фотографии из базы по запросу пользователя;
- рассылкой текста всем подтверждённым пользователям.

## Стек
- Python 3.11
- aiogram 3
- PostgreSQL
- Redis (для FSM)
- psycopg 3 + AsyncConnectionPool
- Docker Compose

## Быстрый старт

1. Скопируй `.env.example` в `.env`
2. Заполни `BOT_TOKEN` и `ADMIN_IDS`
3. Запусти:

```bash
docker compose up --build -d
```

4. Смотреть логи:

```bash
docker compose logs -f bot
```

## Локальный запуск без Docker

Нужны PostgreSQL и Redis.

```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
python run.py
```

## Команды и сценарии

### Пользователь
- `/start` — начало работы
- `/key ВАШ_КЛЮЧ` — активировать доступ на 24 часа
- `/photo` — получить последнюю фотографию из БД

## Как работает фото

1. Админ загружает фотографию.
2. Бот скачивает файл из Telegram.
3. Байты файла сохраняются в PostgreSQL в таблицу `photos`.
4. При запросе пользователя бот берёт последнюю запись из `photos` и отправляет её из базы.

## Что важно

- В базе хранится только `sha256` хеш ключа, не сам ключ.
- Redis используется для FSM, чтобы состояния не терялись после перезапуска.
- PostgreSQL используется и для пользователей, и для ключей, и для доступа, и для фото.
